A Reasonable Secure Login Form

Try logging in with user:password or admin:SecurePa$$w0rd1

Security Features

  • Passwords are securely hashed using Bcrypt.
  • Javascript is not used as it is not necessary. Users of NoScript or a browser that does not support JS, rejoice!
  • Per Bcrypt specification, all passwords must be salted. As such, rainbow table attacks are impossible.
  • All server responses are specifically worded to neither confirm nor deny the existence of accounts, making attacking more difficult.

Possible Improvements

  • What good is a secure password checking system if the password can be sniffed out of the air? Use HTTPS.
  • My implementation is unoptimized and written in pure Python, so it is quite slow. A faster implementation would allow more iterations to be done in the same amount of time, making it harder to crack. While you are at it, make sure that the implementation you use is audited as well.
